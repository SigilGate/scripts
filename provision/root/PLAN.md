# Root-нода: план реализации

Точка входа для пересборки сети с нуля. Root-нода — управляющий узел:
хранит PKI, управляет нодами по SSH, не запускает никаких публичных сервисов.

Аудит проблем в существующих скриптах: [SECURITY_AUDIT.md](./SECURITY_AUDIT.md)

---

## Новые скрипты

### `provision/root/`

| Скрипт | Назначение |
|--------|-----------|
| `root-provision.sh` | Оркестратор. Вызывает общие скрипты и root-специфичные по порядку |
| `setup-firewall-root.sh` | UFW: открыт только порт 22, исходящий SSH unrestricted |
| `setup-workspace.sh` | Клон репозиториев, структура рабочих директорий |

### `pki/`

Скрипты уже существуют, но `ca-init.sh` не охватывает полный bootstrap с нуля.
Нужно убедиться, что PKI можно поднять на чистой ноде без предположений об уже существующем состоянии.

## Переиспользуемые существующие скрипты

Запускаются из оркестратора `root-provision.sh` как есть, после устранения
проблем из аудита:

| Скрипт | Статус | Что исправить перед использованием |
|--------|--------|------------------------------------|
| `provision/common/system-setup.sh` | переиспользуем | автообновление — только security-патчи |
| `provision/common/create-user.sh` | переиспользуем | убрать хардкод пароля (КРИТИЧНО) |
| `provision/common/setup-ssh-key.sh` | переиспользуем | без изменений |
| `provision/common/harden-ssh.sh` | переиспользуем | дополнить: `AllowUsers`, `MaxAuthTries`, `LoginGraceTime`, `X11Forwarding no` |

## Что НЕ запускается на Root

Root-нода не является публичным узлом сети:

- `install-xray.sh`, `configure-xray.sh` — нет
- `install-nginx.sh`, `configure-nginx.sh` — нет
- `setup-tls.sh` — нет (нет публичных сервисов, нет домена)
- `setup-repos.sh` — нет (заменяется на `setup-workspace.sh` без PAT в credentials)

## Порядок выполнения

```
root-provision.sh
│
├── 1. system-setup.sh          # обновление системы, базовые пакеты
├── 2. create-user.sh           # пользователь sigil (после исправления пароля)
├── 3. setup-ssh-key.sh         # SSH-ключ для sigil
├── 4. harden-ssh.sh            # усиление SSH (после дополнений)
├── 5. setup-firewall-root.sh   # НОВЫЙ: только порт 22
├── 6. setup-workspace.sh       # НОВЫЙ: рабочие директории, клон репозиториев
└── 7. pki/ca-init.sh           # инициализация Root CA (SSH CA + структура X.509)
```

После выполнения Root-нода готова к выпуску сертификатов для остальных нод сети.

---

## Принципы, которые закладываем при реализации

- Пароли и секреты — только через переменные окружения, не в аргументах команд
- SSH с других нод проверяется через CA-сертификаты, не `StrictHostKeyChecking=no`
- Приватные ключи PKI не проходят через `/tmp`
- SSH на Root-ноде ограничен по `AllowUsers`; открыт для всего интернета по порту —
  смириться нельзя, рассмотреть ограничение по IP администратора на этапе настройки
